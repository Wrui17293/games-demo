# games-demo

## 基础信息

- 提交人：夏诗宇
- 维护人：夏诗宇
- 来源：互动
- 所属业务域：营销域
- 当前状态：进行中

## 用途

这个 Skill 用于将已确认的 Zero 静态高保真稿转化为可交互、可演示的动态视觉稿，并规划营销互动首页、核心玩法、AI 生成、奖励弹窗、分享页和异常状态的动效方案或演示代码。

## 适用场景

- 高保真稿已经由用户确认，需要进入动态视觉或可运行演示阶段。
- 需要明确每个互动画面的触发条件、动效目的、关键元素、节奏时长、状态变化和可交互要求。
- 需要把营销互动玩法转化为可演示的动效流程或代码实现计划。

## 输入

- 用户已确认的 Zero 静态高保真稿。
- 页面状态、关键交互路径、核心视觉焦点和目标技术栈。
- 需要演示的首页、核心互动页、AI 对话页、游戏页、奖励页、分享页、规则页或异常页。

## 输出

- 动态画面清单。
- 每个动态画面的触发条件、动效描述、节奏时长、状态变化和可交互要求。
- 动效参数确认表。
- 用户确认后的动态视觉稿方案或可运行演示代码规划。

## 依赖资料

- `SKILL.md`：动效判断、输出格式和禁止事项。
- `references/rules.md`：动效触发、节奏、状态变化和边界规则。
- `games-hifi`：已确认的高保真静态稿。
- 用户确认的目标技术栈和演示范围。

## 边界和风险

- 未确认高保真稿前不得进入动态阶段。
- 未确认动效参数前不得生成动态视觉稿或可运行代码。
- 不得所有页面套同一种动效。
- 不得让动效遮挡核心文案或影响主操作完成。
- 动效复杂度、技术栈可行性和上线实现成本需要人工确认。
